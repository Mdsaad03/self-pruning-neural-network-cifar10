# Self-Pruning Neural Network — Experiment Report

## 1. Background

In this project, a feed-forward neural network is used to classify CIFAR-10 images.
The model learns to prune itself **during training** rather than requiring a
separate post-training pruning step. Each weight connection has a learnable gate
score; the sigmoid of this score acts as a soft mask on the weight, and an L1
penalty on the gates encourages the network to push redundant gates toward zero.

This project was completed as part of the **Tredence Analytics AI Engineering
Internship** assessment. All code is contained in `self_pruning_network.py`.

## 2. Why Does L1 Regularisation on Sigmoid Gates Encourage Sparsity?

### 2.1 The Mechanics

For a gate score `s`, the effective gate value is `g = σ(s) ∈ (0, 1)`.
The sparsity loss adds `λ · E[|g|]` to the total loss.

Because `g` is always positive, `|g| = g`, so the gradient of the sparsity
term with respect to `s` is:

```
∂(λ|g|)/∂s = λ · σ(s)(1 − σ(s))
```

This is **always positive**, meaning the sparsity loss *always* pushes gate
scores downward — toward `−∞` — which in turn drives `g` toward `0`.

### 2.2 Why L1 Rather Than L2?

| Property | L1 (Σ\|g\|) | L2 (Σg²) |
|:---|:---:|:---:|
| Gradient near zero | Constant λ pull | Vanishes as g → 0 |
| Tendency | Pushes gates **all the way** to 0 | Merely shrinks gates |
| Resulting distribution | Bimodal (spike at 0 + cluster near 1) | Unimodal, diffuse |

L2 creates a quadratic bowl: the gradient `2g` shrinks as `g` shrinks, so the
optimiser effectively stops pushing once a gate is "small enough." L1's constant
gradient means the cost of keeping a gate alive is independent of its current
magnitude — useless gates get pruned to true zero while useful ones resist the pressure.

### 2.3 Intuition

Think of each gate as an employee on a flat salary of `λ` per unit of activity.
If an employee (weight) does not generate enough benefit to offset their salary
(classification loss reduction), the optimiser fires them (gate → 0). Employees
who are genuinely useful negotiate a raise (the classification gradient counters
the sparsity pull) and stay active.

## 3. Architecture

```
Input (32×32×3) → Flatten → 3,072
  └── PrunableLinear(3072 → 1024) + BatchNorm1d + ReLU + Dropout(0.3)
        └── PrunableLinear(1024 → 256)  + BatchNorm1d + ReLU + Dropout(0.3)
              └── PrunableLinear(256  → 10)   [logits]
```

Each `PrunableLinear` layer holds three learnable parameter tensors:

| Parameter | Shape | Role |
|:---|:---|:---|
| `weight` | `(out, in)` | Standard weight matrix |
| `bias` | `(out,)` | Standard bias vector |
| `gate_scores` | `(out, in)` | Learned pruning masks (pre-sigmoid) |

**Forward pass:**

```python
gates          = sigmoid(gate_scores)    # soft gates ∈ (0, 1)
pruned_weights = weight * gates          # element-wise mask
output         = pruned_weights @ x.T + bias
```

## 7. Observations and Takeaways

1. **Higher λ → more sparsity, lower accuracy (trade-off).**
   A very large λ over-prunes the network, removing connections that were still
   useful and degrading test accuracy.

2. **Moderate λ (≈ 0.001) typically hits the sweet spot**, achieving meaningful
   compression without significant accuracy degradation.

3. **The bimodal gate distribution confirms the L1 mechanism is working.**
   Gates polarise toward 0 or 1 rather than shrinking uniformly — the hallmark
   of true sparse pruning rather than weight decay.

4. **Self-pruning is more efficient than post-training pruning.** Because the
   network adapts its topology throughout training rather than after, it can
   find better sparse solutions: the remaining weights compensate for removed
   ones in real time.

5. **BatchNorm + Dropout complement the sparsity pressure.** BatchNorm stabilises
   gradient flow through layers where many weights are gated to near-zero; Dropout
   prevents the surviving connections from co-adapting too tightly.
